📖 什么是 Serverless？

Serverless 不是"没有服务器"，而是你不需要管理服务器。你只需编写函数代码， 云平台负责在请求到来时自动分配资源、执行代码、并自动扩展。

传统服务器Serverless
部署单位整个应用单个函数
扩缩容手动/自动，分钟级自动，毫秒级
计费按时间（即使无请求）按请求量
冷启动无（常驻进程）首次请求数 ms 延迟
运维OS/依赖/安全更新零运维
运行位置指定机房全球边缘节点

🏗 Cloudflare Workers 架构

Workers 运行在 V8 JavaScript 引擎的 Isolate 中（不是容器/虚拟机）。 每个 Isolate 仅占用 ~5MB 内存，启动时间 <5ms——这就是为什么 Workers 的"冷启动"几乎不可感知。

请求到达边缘节点
├─ Isolate 已存在（热启动）→ 立即执行（<1ms）
└─ Isolate 不存在（冷启动）→ 创建新 Isolate（~5ms）→ 加载代码 → 执行

对比 Lambda 冷启动：容器启动 200ms-2s
Workers 冷启动快 40-400 倍

💡 Worker 的关键特性

V8 Isolate

不是 Docker 容器！每个 Worker 实例在一个 V8 Isolate 中运行， 启动速度极快（~5ms），内存占用极低（~5MB）。

全球分布式

代码自动部署到 Cloudflare 全球 330+ 节点。 用户请求由最近的节点处理，天然低延迟。

Request/Response API

Workers 使用标准的 Fetch API（Request/Response）， 与浏览器中的 fetch 用法一致，学习成本极低。

免费额度

每天 10 万次请求免费，完全满足个人项目和学习用途。 无需信用卡即可开始。
