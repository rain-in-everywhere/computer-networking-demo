🔍 DNS 解析

通过 DNS-over-HTTPS (DoH) 加密查询域名的 DNS 记录，理解互联网的"电话簿"

📖 DNS：互联网的电话簿

DNS（Domain Name System）将人类可读的域名（如 google.com）翻译为机器可读的 IP 地址（如 142.250.80.46）。 每次你打开一个网页，第一步就是 DNS 查询。

DNS 查询层次（递归解析）：
① 本地缓存 → ② 根域名服务器 → ③ .com TLD 服务器 → ④ 权威 DNS → ⑤ 返回 IP

🧪 实验：DNS 查询

输入域名，选择记录类型，调用 /api/dns-lookup 进行 DNS-over-HTTPS (DoH) 加密查询。 注意：DNS 查询结果也可能被 CDN 或上游缓存影响。

📋 DNS 记录类型速查

A 记录

域名 → IPv4 地址。
例：example.com → 93.184.216.34

AAAA 记录

域名 → IPv6 地址。
例：example.com → 2606:2800:220:1:248:1893:25c8:1946

CNAME 记录

域名别名。将 www 指向主域名，或指向 CDN 提供的域名。

MX 记录

邮件交换服务器。指定接收该域名邮件的服务器及优先级。

TXT 记录

文本记录。常用于 SPF/DKIM 邮件验证、域名所有权验证。

DoH 加密

DNS-over-HTTPS 将 DNS 查询封装在 HTTPS 中，防止窃听和篡改。
